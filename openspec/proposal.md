# 需求规格说明书 - SD-01 RESTful 任务管理 API

## 1. 项目概述

### 1.1 背景

本项目旨在构建一个轻量级 RESTful 任务管理 API，为上层应用提供统一、规范的任务数据操作接口。通过标准化的 CRUD 接口、灵活的多条件过滤与分页排序能力，支持各类客户端对任务数据进行高效管理。

### 1.2 目标

- 提供完整的任务资源生命周期管理（创建、查询、更新、删除）。
- 支持通过多维度条件（状态、优先级、时间范围等）检索任务。
- 提供一致的分页、排序与查询参数接口。
- 通过自动生成的 OpenAPI/Swagger 文档降低接入成本。
- 通过统一的数据验证与错误响应机制保障接口的健壮性。

### 1.3 范围

**包含范围：**
- 任务资源的 CRUD 接口设计与实现。
- 查询接口的多条件过滤、分页、排序参数规格。
- 请求数据的输入验证与错误响应格式。
- 基于 OpenAPI 3.1 的 API 文档自动生成。

**不包含范围：**
- 用户认证与授权（本版本为单用户/开放 API）。
- 前端界面开发。
- 文件附件上传功能。
- 任务标签/分类管理。
- 任务指派/协作功能。

## 2. 功能需求

### 2.1 用户角色

| 角色名称 | 描述 | 权限 |
|---------|------|------|
| API 消费者 | 调用本 API 的客户端应用或开发者 | 对所有任务资源拥有完整的 CRUD 权限 |


### 2.2 功能清单

#### 2.2.1 任务 CRUD（FR-001 ~ FR-004）

##### FR-001 创建任务

- **需求 ID**: FR-001
- **需求描述**: 客户端通过 HTTP POST 请求向 `/api/v1/tasks` 提交任务数据，系统校验通过后持久化并返回创建成功的任务对象。
- **优先级**: 高
- **验收标准**:
  - 接口接收 JSON 格式的请求体。
  - 必填字段缺失时返回 422 错误。
  - 字段值非法时返回 422 错误并指明具体字段与原因。
  - 创建成功后返回 HTTP 201，响应体包含完整的任务对象（含系统生成的 `id`、`created_at`、`updated_at`）。
- **依赖关系**: 无

##### FR-002 获取任务详情

- **需求 ID**: FR-002
- **需求描述**: 客户端通过 HTTP GET 请求访问 `/api/v1/tasks/{task_id}`，系统返回对应 ID 的任务详情。
- **优先级**: 高
- **验收标准**:
  - 任务存在时返回 HTTP 200 及任务对象。
  - 任务不存在时返回 HTTP 404，并提供明确的错误信息。
  - `task_id` 格式非法时返回 HTTP 422。
- **依赖关系**: FR-001

##### FR-003 更新任务

- **需求 ID**: FR-003
- **需求描述**: 客户端通过 HTTP PUT 或 PATCH 请求向 `/api/v1/tasks/{task_id}` 提交更新数据，系统校验后更新对应任务。
- **优先级**: 高
- **验收标准**:
  - 支持全量更新（PUT）与部分更新（PATCH）。
  - 更新成功后返回 HTTP 200 及更新后的完整任务对象。
  - `updated_at` 字段在每次成功更新后自动刷新为当前时间。
  - 对不存在的任务返回 HTTP 404。
  - 对非法字段值返回 HTTP 422 并指明具体字段与原因。
- **依赖关系**: FR-001, FR-002


##### FR-004 删除任务

- **需求 ID**: FR-004
- **需求描述**: 客户端通过 HTTP DELETE 请求访问 `/api/v1/tasks/{task_id}`，系统删除对应任务。
- **优先级**: 高
- **验收标准**:
  - 任务存在时执行删除并返回 HTTP 204（无响应体）。
  - 任务不存在时返回 HTTP 404。
  - `task_id` 格式非法时返回 HTTP 422。
- **依赖关系**: FR-001, FR-002


#### 2.2.2 任务列表查询与过滤（FR-005）

##### FR-005 获取任务列表（支持过滤、分页、排序）

- **需求 ID**: FR-005
- **需求描述**: 客户端通过 HTTP GET 请求访问 `/api/v1/tasks`，系统返回符合条件的任务列表。该接口支持多条件组合过滤、分页与排序。
- **优先级**: 高
- **验收标准**:
  - 无过滤条件时返回所有任务（受分页限制）。
  - 支持同时应用多个过滤条件，条件之间为 **AND** 关系。
  - 支持分页参数 `page` 与 `page_size`。
  - 支持排序参数 `sort_by` 与 `sort_order`。
  - 返回响应中包含分页元数据（总记录数、当前页、每页大小、总页数）。
- **依赖关系**: FR-001


#### 2.2.3 数据验证与错误处理（FR-006）

##### FR-006 统一的数据验证与错误响应

- **需求 ID**: FR-006
- **需求描述**: 系统对所有入参请求进行数据校验，并在校验失败或业务异常时返回结构统一、语义清晰的错误响应。
- **优先级**: 高
- **验收标准**:
  - 请求体或查询参数格式非法时返回 HTTP 422 Unprocessable Entity。
  - 资源不存在时返回 HTTP 404 Not Found。
  - 所有错误响应体包含 `detail` 字段（或数组），描述错误原因。
  - 字段级校验错误需指明具体字段名与失败原因。
- **依赖关系**: 无

#### 2.2.4 API 文档自动生成（FR-007）

##### FR-007 OpenAPI/Swagger 文档

- **需求 ID**: FR-007
- **需求描述**: 系统基于接口定义自动生成符合 OpenAPI 3.1 规范的 API 文档，并提供可交互的 Swagger UI。
- **优先级**: 中
- **验收标准**:
  - 文档可通过 `/api/v1/docs`（或框架默认路径）访问 Swagger UI。
  - 文档可通过 `/api/v1/openapi.json` 获取原始 JSON 定义。
  - 文档中包含所有端点、请求/响应模型、字段类型、必填标记、枚举值说明。
  - 文档自动随代码变更同步更新。
- **依赖关系**: 无

## 3. 数据需求

### 3.1 任务模型字段定义

| 字段名 | 数据类型 | 必填 | 默认值 | 约束与说明 |
|--------|---------|------|--------|-----------|
| `id` | Integer (主键) | 系统自动生成 | 自增 | 唯一标识，不可修改 |
| `title` | String | 是 | — | 任务标题；最大长度限制（建议 1~200 字符） |
| `description` | String | 否 | `null` | 任务详细描述；最大长度限制（建议 2000 字符） |
| `status` | String (枚举) | 是 | `"todo"` | 任务状态；枚举值建议：`todo`（待办）、`in_progress`（进行中）、`done`（已完成） |
| `priority` | String (枚举) | 是 | `"medium"` | 任务优先级； 枚举值建议：`low`（低）、`medium`（中）、`high`（高 |
| `due_date` | DateTime (ISO 8601) | 否 | `null` | 任务截止时间；格式建议 `YYYY-MM-DDTHH:MM:SS`|
| `created_at` | DateTime | 系统自动生成 | 当前时间 | 创建时间戳，不可修改 |
| `updated_at` | DateTime | 系统自动生成 | 当前时间 | 最后更新时间戳，每次更新自动刷新 |


### 3.2 过滤参数规格

获取任务列表接口（`GET /api/v1/tasks`）支持的查询参数：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `status` | String | 否 | 按状态过滤；支持单一值或逗号分隔的多值（如 `todo,in_progress`） |
| `priority` | String | 否 | 按优先级过滤；支持单一值或逗号分隔的多值（如 `low,medium`） |
| `due_date_from` | DateTime (ISO 8601) | 否 | 截止时间起始范围（包含） |
| `due_date_to` | DateTime (ISO 8601) | 否 | 截止时间结束范围（包含） |
| `created_at_from` | DateTime (ISO 8601) | 否 | 创建时间起始范围（包含） |
| `created_at_to` | DateTime (ISO 8601) | 否 | 创建时间结束范围（包含） |
| `search` | String | 否 | 全文搜索关键词，对 `title` 与 `description` 进行模糊匹配 |


### 3.3 分页参数规格

| 参数名 | 类型 | 必填 | 默认值 | 约束 |
|--------|------|------|--------|------|
| `page` | Integer | 否 | `1` | 当前页码，最小值为 `1` |
| `page_size` | Integer | 否 | `20` | 每页记录数；最大值限制为 `100` |

**分页响应元数据（建议在响应体中独立封装）：**

```json
{
  "items": [...],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 150,
    "total_pages": 8
  }
}
```

### 3.4 排序参数规格

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `sort_by` | String | 否 | `"created_at"` | 排序字段；支持：`title`、`status`、`priority`、`due_date`、`created_at`、`updated_at` |
| `sort_order` | String | 否 | `"desc"` | 排序方向；支持：`asc`（升序）、`desc`（降序） |

## 4. 接口规格

### 4.1 端点一览

| 方法 | 路径 | 功能 | 成功状态码 |
|------|------|------|-----------|
| POST | `/api/v1/tasks` | 创建任务 | 201 |
| GET | `/api/v1/tasks` | 获取任务列表 | 200 |
| GET | `/api/v1/tasks/{task_id}` | 获取任务详情 | 200 |
| PUT/PATCH | `/api/v1/tasks/{task_id}` | 更新任务 | 200 |
| DELETE | `/api/v1/tasks/{task_id}` | 删除任务 | 204 |

### 4.2 错误响应规格

所有错误响应应返回结构统一的 JSON 对象，便于客户端解析与处理。

**通用错误响应体结构：**

```json
{
  "detail": "人类可读的错误描述"
}
```

**字段级校验错误（HTTP 422）响应体结构：**

```json
{
  "detail": [
    {
      "loc": ["body", "title"],
      "msg": "field required",
      "type": "value_error.missing"
    }
  ]
}
```

**核心错误场景映射：**

| 场景 | HTTP 状态码 | 响应内容说明 |
|------|------------|-------------|
| 请求体 JSON 格式错误 | 400 | 提示请求体无法解析 |
| 字段级数据校验失败 | 422 | 列出所有非法字段及原因 |
| 任务资源不存在 | 404 | 指明对应 `task_id` 的任务未找到 |
| 分页参数越界（如 `page` < 1） | 422 | 提示参数范围非法 |
| 排序字段不支持 | 422 | 提示 `sort_by` 字段不在允许列表中 |

## 5. 用户故事

### US-001 创建新任务

**作为** API 消费者  
**我想要** 通过发送包含标题、描述、状态、优先级和截止时间的数据来创建新任务  
**以便于** 在系统中记录待办事项

**验收条件**:
- 提供合法的 `title` 后，系统成功创建任务并返回唯一 `id`。
- 缺少 `title` 时，系统拒绝创建并返回 422 错误。
- 非法的 `status` 或 `priority` 值被拒绝并返回 422 错误。

### US-002 查看任务列表并筛选

**作为** API 消费者  
**我想要** 获取任务列表，并按状态、优先级或时间范围进行过滤  
**以便于** 快速定位我关心的任务

**验收条件**:
- 无过滤条件时返回所有任务（分页）。
- 同时指定 `status=todo` 和 `priority=high` 时，仅返回同时满足两个条件的任务。
- 结果按指定的字段和方向排序。

### US-003 更新任务状态

**作为** API 消费者  
**我想要** 修改已有任务的内容或状态  
**以便于** 跟踪任务的最新进展

**验收条件**:
- 成功更新后，`updated_at` 字段自动更新。
- 对不存在的任务 ID 返回 404 错误。

### US-004 删除已完成任务

**作为** API 消费者  
**我想要** 删除不再需要的任务  
**以便于** 清理任务列表

**验收条件**:
- 删除成功后返回 204 无内容。
- 再次访问该任务 ID 返回 404。

## 6. 假设和依赖

### 6.1 假设

- 当前版本为单用户系统，不考虑多租户或用户隔离。
- 所有日期时间字段在接口层面统一采用 ISO 8601 格式字符串交互。
- API 版本通过 URL 路径 `/api/v1/` 进行硬编码管理，不存在版本协商机制。

### 6.2 依赖

- 依赖后端框架 FastAPI 提供的请求校验、依赖注入与自动文档生成能力。
- 依赖 SQLAlchemy 2.0 异步会话来操作 SQLite 数据库。
- 依赖 Pydantic v2 进行请求/响应模型的序列化与反序列化。
